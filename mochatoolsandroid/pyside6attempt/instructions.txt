# Mocha Tools Android: Ubuntu/PySide6 build notes

These steps are for Ubuntu. `pyside6-android-deploy` currently needs a Unix host, and the virtualenv should stay outside the project folder so it is not packaged into the APK.

## 1. Create the Python virtualenv and test imports

```bash
cd /path/to/mochatoolsandroid/pyside6attempt
bash setup_and_build.sh
source ~/mochatools-android-env/bin/activate
python main.py
```

Resize the desktop window to about 390x844 for a quick portrait-phone check.

## 2. Install Qt for Android

Install Qt Online Installer from:

```text
https://www.qt.io/download-qt-installer
```

In the installer, select:

- Qt 6.7 or newer
- Desktop Qt for your Ubuntu host, for example `gcc_64`
- Qt for Android: `android_arm64_v8a`
- Android SDK Platform API 34
- Android SDK Build-Tools 34.0.0
- Android NDK r26b / `26.1.10909125`

The script can also install the Android command-line SDK/NDK pieces:

```bash
bash setup_and_build.sh --android-tools
```

Use Qt Creator afterward to confirm the Android kit sees:

```bash
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export ANDROID_SDK_ROOT=$HOME/Android/Sdk
export ANDROID_NDK_ROOT=$HOME/Android/Sdk/ndk/26.1.10909125
export PATH=$ANDROID_SDK_ROOT/platform-tools:$ANDROID_SDK_ROOT/cmdline-tools/latest/bin:$PATH
```

## 3. Create Qt for Python Android wheels

`pyside6-android-deploy` needs Android-target PySide6 and shiboken6 wheels. There are two ways to get them.

For open-source/no-commercial-license builds, cross-compile them from source:

```bash
source ~/mochatools-android-env/bin/activate
export QT_INSTALL_PATH=$HOME/Qt/6.11.1
bash setup_and_build.sh --oss-wheels
```

`QT_INSTALL_PATH` must point at the Qt version folder that contains `android_arm64_v8a`. Adjust `6.11.1` to the version installed by Qt Maintenance Tool.

For commercial Qt accounts, `qtpip` can download prebuilt wheels:

```bash
source ~/mochatools-android-env/bin/activate
bash setup_and_build.sh --wheels
```

On Ubuntu 26.04, `qtpip` may need a local compatibility path for `libxml2.so.2`; the script creates it under `~/mochatools-android-env/qtpip-compat-libs` without touching system libraries. If `qtpip` prints `Commercial License NOT found`, use `--oss-wheels`.

The OSS wheel builder uses Qt's official source workflow:

```bash
git clone https://code.qt.io/pyside/pyside-setup
python tools/cross_compile_android/main.py --plat-name=aarch64 --qt-install-path=$QT_INSTALL_PATH --auto-accept-license --skip-update
```

After either path succeeds, the script rewrites `pysidedeploy.spec` with the real `python_path`, `wheel_pyside`, `wheel_shiboken`, `sdk_path`, and `ndk_path`.

## 4. Build the Android package

```bash
source ~/mochatools-android-env/bin/activate
export QT_INSTALL_PATH=$HOME/Qt/6.11.1
bash setup_and_build.sh --build
```

Debug mode produces an APK. Release mode in `pysidedeploy.spec` produces an AAB.

Install/test:

```bash
adb install path/to/app.apk
adb logcat | grep -E "(python|PySide|mochatools|MochaTools)"
```

## 5. One-command fresh Ubuntu setup

```bash
cd /path/to/mochatoolsandroid/pyside6attempt
bash setup_and_build.sh --all
```

If Qt Online Installer put SDK/NDK somewhere else, set `ANDROID_SDK_ROOT` and `ANDROID_NDK_ROOT` before running the script.
